Lab Architecture

In this lab we have setup the following infrastructure

Application VPC (AWS VPC Main)


  • The application VPC has an EC2 instance running a vulnerable web application inside a docker container.
  • The container is listening on port 8080.
  • An AWS ELB exposes the application to the internet on port 80
  • AWS WAF is configured with the AWS OWASP Top 10 ruleset.
  • Security Groups are configured to block all ports inbound from the internet other than port 80
  • The EC2 instance has no external IP and outbound external connectivity is provided via a NAT gateway


Attacker VPC (AWS VPC Kali)


  • The attacker EC2 instance running this web application together with some tools to attack the vulnerable application.
  • The attacker instance has a public ip and connects to the web application over public internet via the AWS ELB public IP.
